    Probability Distributions తో Call Center Operations ని మోడల్ చేయడం¶

    Call centers ఒక డైనమిక్ ఎన్విరాన్‌మెంట్‌లో పనిచేస్తాయి, ఇక్కడ:
    1. కస్టమర్ కాల్స్ రోజంతా యాదృచ్ఛికంగా వస్తాయి.
    2. ఏజెంట్లు వివిధ success రేట్లతో సమస్యలను పరిష్కరిస్తారు.
    3. Waiting time ట్రాఫిక్ మరియు స్టాఫింగ్ ఆధారంగా మారుతాయి.

    ఈ లక్షణాలు call centres ని probability distributions ని ఉపయోగించడానికి అద్భుతమైన వాస్తవ ప్రపంచ ఉదాహరణగా మారుస్తాయి. call arrivals, waiting times, మరియు పరిష్కార నమూనాలను మోడల్ చేయడం ద్వారా, మనం:
    1. పని లోడ్ మరియు స్టాఫింగ్ అవసరాలను అంచనా వేయవచ్చు.
    2. సేవా స్థాయిలను మెరుగుపరచవచ్చు.
    3. కస్టమర్ అనుభవ మెట్రిక్‌లను అర్థం చేసుకోవచ్చు.

    ఈ బ్లాగ్‌లో, queueing theory మరియు service analytics లో విస్తృతంగా ఉపయోగించే మూడు ముఖ్యమైన distributions పై మనం దృష్టి సారిస్తాము:
    1. Poisson Distribution – నిర్దిష్ట కాల వ్యవధిలో వచ్చే కాల్స్ సంఖ్యను మోడల్ చేయడానికి.
    2. Exponential Distribution – వరుస కాల్స్ మధ్య సమయాన్ని మోడల్ చేయడానికి.
    3. Geometric Distribution – విజయవంతమైన పరిష్కారం కంటే ముందు విఫలమైన ప్రయత్నాల సంఖ్యను మోడల్ చేయడానికి.

    తర్వాత, Goodness-of-Fit tests ని పరిశీలించి Central Limit Theorem ని ధృవీకరించacağız.

    if pd.isna(s): return np.nan
    try:
        h, m, s2 = 0, 0, 0
        parts = str(s).split(':')
        parts = [int(p) for p in parts]
        if len(parts) == 3: h, m, s2 = parts
        elif len(parts) == 2: m, s2 = parts
        else: return float(parts[0])
        return h*3600 + m*60 + s2

    Poisson distribution (calls/day)¶

    “తర్వాతి గంటలో ఎంత కాల్స్ వస్తాయి?” అనే ప్రశ్నను పరిగణనలోకి తీసుకుందాం.
    కాల్ వస్తువులు సాధారణంగా స్వతంత్రంగా ఉంటాయి మరియు చిన్న వ్యవధులలో సగటు రేటు స్థిరంగా ఉంటాయి. Poisson distribution ఇక్కడ ప్రత్యేకంగా ఉపయోగకరంగా ఉంటుంది.
    Definition
    Poisson distribution ఒక నిర్దిష్ట కాల వ్యవధిలో \(k\) ఈవెంట్లను గమనించే probability ని మోడల్ చేస్తుంది, ఈ ఈవెంట్లు స్వతంత్రంగా ఒక స్థిరమైన రేటు \(\lambda\) వద్ద జరుగుతాయి.

    \[P(X=x)= \frac{\lambda^xe^{-\lambda}}{x!}\]
    # Calculate: counts/hour
hourly_counts = df.set_index('timestamp').groupby([pd.Grouper(freq='h'), 'Agent']).size().reset_index()

# Create a dataset that has all combination of agent-timestamp (hours)

# Metrics

ax.set_xlabel('Calls per hour')
ax.legend()
plt.show();

    Exponential distribution (inter-arrival times)¶

    ఇప్పుడు, “తర్వాతి కాల్ వస్తే ఎంత సమయం పడుతుంది?” అనే ప్రశ్నకు సమాధానం ఇవ్వండి.
    కాల్‌లు Poisson ప్రక్రియను అనుసరిస్తే, వరుస కాల్స్ మధ్య సమయం మేమరీలెస్, అంటే, మీరు ఎంత సమయం వేచి ఉన్నారో అది తర్వాతి కాల్ వచ్చే probability ని ప్రభావితం చేయదు. ఇది Exponential distribution ద్వారా మోడల్ చేయబడుతుంది.
    Definition Exponential distribution Poisson ప్రక్రియలో ఈవెంట్ల మధ్య సమయాన్ని మోడల్ చేస్తుంది: $$ PDF:= f(\lambda) = \lambda\times e^{-\lambda t} $$
    ఇక్కడ:
    \(t\) = వేచి ఉండే సమయం (ఉదాహరణకు, కాల్స్ మధ్య నిమిషాలు),
    \(\lambda\) = రేటు ప్యారామీటర్ (నిమిషానికి కాల్స్).

    Key Properties 1. సగటు వేచి ఉండే సమయం = \(1/\lambda\). 2. మేమరీలెస్ ప్రాపర్టీ: $ P(T > s + t \mid T > s) = P(T > t)$

    Example
    కాల్‌ల మధ్య సగటు సమయం 53 నిమిషాలు అయితే:
    $$ \hat{\lambda} = \frac{1}{50} = 0.02 \text{ per minute} $$ తర్వాతి కాల్ 10 నిమిషాల లోపు వస్తే probability:
    $$ P(T \le 10) = 1 - e^{-0.02 \times 10} \approx 0.18 $$

    # Exponential: inter-arrival times
ts = df['timestamp'].sort_values().values
inter_arrival_sec = np.diff(ts) / np.timedelta64(1, 's')
inter_arrival_sec = inter_arrival_sec[inter_arrival_sec > 0]
inter_arrival_min = inter_arrival_sec / 60.0

# Metrics

fig, ax = plt.subplots(figsize=(10,6))
sns.histplot(inter_arrival_min, bins=40, stat='density', color="gold", ax=ax)
x = np.linspace(0, np.percentile(inter_arrival_min, 99), 200)
ax.plot(x, stats.expon.pdf(x, scale=1/lambda_exp_hat), color="green", lw=2, 
        label=f"Expon PDF (λ={lambda_exp_hat:.3f} PDF)")
ax.set_title('Inter-arrival Times (minutes) with Exponential Fit')

    Geometric Distribution (failures before first success)¶

    “ఒక సమస్య పరిష్కరించబడే ముందు ఎంత విఫలమైన ప్రయత్నాలు జరుగుతాయి?” అనే ప్రశ్నను పరిగణించండి.
    Call centres లో, కొన్ని సమస్యలు విజయానికి ముందు అనేక ఇంటరాక్షన్లను అవసరం చేస్తాయి. ప్రతి ప్రయత్నం Bernoulli trial గా మోడల్ చేయబడుతుంది: లేదా విజయవంతం లేదా విఫలమవుతుంది.
    Definition
    Geometric distribution మొదటి విజయానికి ముందు విఫలమైన సంఖ్యను మోడల్ చేస్తుంది:
    $$ P(X=r)=p\times (1-p)^{r} $$
    ఇక్కడ r = 0,1,2,3,4,...
    \(p\) = ఏదైనా ప్రయత్నంలో విజయానికి probability
    \(r\) = విజయానికి ముందు విఫలమైన సంఖ్య
    Example ఒక ప్రయత్నంలో సమస్యను పరిష్కరించే probability 0.73 అయితే, విజయానికి ముందు 2 విఫలాలు ఉండే probability:
    $$ P(X = 2) = (1 - 0.73)^2 \times 0.73 $$

    Insights 1. p యొక్క అధిక విలువ ఎక్కువగా సమస్యలు త్వరగా పరిష్కరించబడతాయి అని సూచిస్తుంది.
    2. తక్కువ p సంక్లిష్టమైన కేసులు లేదా శిక్షణ లోపాలను సూచిస్తుంది. 

    # Failures before first success (per agent)

# For each agent’s chronological sequence of resolved flags, count the number of N (failures) before the next Y (first success).
geo_failures = []
for agent, grp in df.sort_values('timestamp').groupby('Agent'):
    seq = grp['Resolved'].dropna().values
    fail = 0
    for v in seq:
        if v == 1:
            geo_failures.append(fail)
            fail = 0
        elif v == 0:
            fail += 1

geo_failures = np.array(geo_failures)

# Fit a Geometric distribution on “failures before success” across agents; 
# estimate success probability p (MLE via mean failures).
if len(geo_failures) > 0:
    mean_fail = float(np.mean(geo_failures))
    p_hat = 1.0 / (mean_fail + 1.0)
    k_max_g = int(np.percentile(geo_failures, 99))
    k_vals_g = np.arange(0, max(5, k_max_g)+1)
    pmf_geom = stats.geom.pmf(k_vals_g+1, p_hat)

    fig, ax = plt.subplots(figsize=(10,6))
    sns.histplot(geo_failures, bins=range(0, k_vals_g[-1]+2), stat='probability', 
                 color="gold", ax=ax)
    ax.plot(k_vals_g, pmf_geom, 'o-', color="green", label=f"Geom(p={p_hat:.3f}) PMF (shifted)")
    ax.set_title('Failures before First Success vs Geometric PMF')
    ax.set_xlabel('Failures before success')
    ax.legend()
    plt.show();

    3. డేటా ఆధారిత నిర్ణయాల ద్వారా కస్టమర్ సంతృప్తిని పెంచవచ్చు.

    Staffing and Scheduling¶

    Poisson arrival rates గంటకు అంచనా వేయడానికి సహాయపడతాయి. ఇది ఏజెంట్ షిఫ్టులను ప్లాన్ చేయడానికి మరియు అండర్- లేదా ఓవర్-స్టాఫింగ్ ని నివారించడానికి ఉపయోగించండి.

    Service Level Agreements (SLAs)¶

    Exponential waiting times మాకు దీర్ఘ వేచి ఉండే probability ని అంచనా వేయడానికి అనుమతిస్తాయి. ఉదాహరణకు, SLA 30 సెకన్లలో సమాధానం ఇవ్వాలని కోరుకుంటే, ఈ క్రింది విధంగా లెక్కించండి:
    $$ P(T \le 0.5 \text{ min}) = 1 - e^{-\lambda \times 0.5} $$

    Validating Distribution Fits with Chi-Square Tests¶

    ఒక సిద్ధాంత distribution ని వాస్తవ ప్రపంచ డేటాకు సరిపోల్చేటప్పుడు, మోడల్ గమనించిన ఫ్రీక్వెన్సీలను ఎంత బాగా వివరిస్తుందో తనిఖీ చేయాలి. ఒక సాధారణ పద్ధతి Chi-Square Goodness-of-Fit (GOF) test.
    ఈ పరీక్ష:
    1. గమనించిన ఫ్రీక్వెన్సీలు (మీ డేటాసెట్ నుండి).
    2. అంచనా వేయబడిన ఫ్రీక్వెన్సీలు (ఫిట్టెడ్ distribution నుండి).

    పరీక్షా గణాంకం: $$ \chi^2 = \sum_{i=1}^{k} \frac{(O_i - E_i)^2}{E_i}$$ ఇక్కడ: \(O_i\) = బిన్ \(i\) లో గమనించిన సంఖ్య,
    \(E_i\) = బిన్ \(i\) లో అంచనా వేయబడిన సంఖ్య,
    \(k\) = బిన్ల సంఖ్య.

    Interpretation:

    Null hypothesis \(H_0\): డేటా నిర్దిష్ట distribution ని అనుసరిస్తుంది.
    పెద్ద \(\chi^2\) → చెత్త సరిపోల్చడం.
    p-value > 0.05 → \(H_0\) ని తిరస్కరించలేము (మంచి సరిపోల్చడం).

    Chi-Square GOF కోసం దశలు
    1. డేటాను బిన్ చేయండి (5 కంటే తక్కువ అంచనా సంఖ్య ఉన్న బిన్లను కలపండి).
    2. గమనించిన మరియు అంచనా వేయబడిన ఫ్రీక్వెన్సీలను లెక్కించండి.
    3. Python లో scipy.stats.chisquare() ని ఉపయోగించండి.

    Central Limit Theorem¶

    Central Limit Theorem చెబుతుంది:

    మీరు ఏ జనాభా నుండి పునరావృత నమూనాలను తీసుకుంటే మరియు వాటి సగటులను లెక్కిస్తే, ఆ నమూనా సగటుల యొక్క distribution నమూనా పరిమాణం పెరిగేకొద్దీ ఒక సాధారణ distribution కి చేరుకుంటుంది—అసలు జనాభా యొక్క ఆకారాన్ని పరిగణనలోకి తీసుకోకుండా.

    ఇది call centres కి ఎందుకు ముఖ్యమైనది?
    Call centre metrics వంటి కాల్ వస్తువుల సంఖ్య లేదా ప్రతి ఏజెంట్ యొక్క సగటు మాట్లాడే వ్యవధి సాధారణంగా skewed లేదా discrete distributions (Poisson, Exponential, etc.) ని కలిగి ఉంటాయి. కానీ మేనేజర్లు రోజువారీ లేదా వారానికి సగటులను చూస్తే, ఆ సగటులు సాధారణంగా CLT వల్ల సాధారణంగా కనిపిస్తాయి.

    Example: Calls per Hour
    వ్యక్తిగత గంటల కాల్ సంఖ్యలు Poisson distribution ని అనుసరిస్తాయి (skewed, discrete). 30 గంటల నమూనాలను తీసుకుని, వాటి సగటు కాల్ సంఖ్యను లెక్కిస్తే, మరియు ఇది చాలా సార్లు పునరావృతం చేస్తే:
    1. ఈ నమూనా సగటుల యొక్క హిస్టోగ్రామ్ బెల్-ఆకారంలో ఉంటుంది.
    2. ఇది అసలు గంటల సంఖ్యలు సాధారణంగా ఉండకపోయినప్పటికీ జరుగుతుంది.

    Visual Demonstration
    1. గంటల కాల్ సంఖ్యల డేటాసెట్ తో ప్రారంభించండి.
    2. 1,000 నమూనాలను డ్రా చేయండి, ప్రతి ఒక్కటి \(n = 30\) గంటల పరిమాణం.
    3. ప్రతి నమూనా కోసం గంటకు కాల్స్ యొక్క సగటును లెక్కించండి.
    4. ఈ సగటుల యొక్క హిస్టోగ్రామ్ ని ప్లాట్ చేయండి, ఇది సాధారణ distribution ని అంచనా వేస్తుంది.
