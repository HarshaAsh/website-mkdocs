    Two‑Proportion Z‑Test : Marketing A/B Conversion¶

    డిజిటల్ మార్కెటింగ్‌లో, కంపెనీలు తరచుగా A/B టెస్టులను నిర్వహించి వెబ్‌పేజీ లేదా ప్రచారానికి రెండు వెర్షన్లను పోలిస్తాయి. లక్ష్యం ఏ వెర్షన్ ఎక్కువ కన్వర్షన్లకు (ఉదాహరణకు, కొనుగోళ్లు, సైన్-అప్‌లు) దారితీస్తుందో తెలుసుకోవడం. ఈ డేటాసెట్‌లో, కంట్రోల్ గ్రూప్ పాత లాండింగ్ పేజీని చూశారు, కాగా ట్రీట్మెంట్ గ్రూప్ కొత్త పేజీని చూశారు. హైపోథసిస్ టెస్ట్ కొత్త పేజీ కన్వర్షన్ రేట్లను ముఖ్యంగా మెరుగుపరుస్తుందో లేదో తనిఖీ చేస్తుంది.

    ఫలితాల వివరణ: Two-proportion Z-test నడిపించిన తర్వాత, మేము p-value పొందాము.

    ఈ p-value 0.05 కంటే ఎక్కువ అయితే, మేము null hypothesis ను తిరస్కరించలేము, అంటే పాత మరియు కొత్త పేజీల మధ్య గణాంకపరమైనంగా ముఖ్యమైన తేడా లేదు. ఇది పునరావృతం చేయడం ద్వారా కొలిచే మెరుగుదల లేదు అని సూచిస్తుంది.
    p-value 0.05 కంటే తక్కువ అయితే, కొత్త పేజీ పాత పేజీ కంటే వేరుగా (మంచి లేదా చెడు) పనిచేస్తుందని మేము తేలిస్తాము.

    ఈ అవగాహన మార్కెటింగ్ బృందాలకు కొత్త డిజైన్‌ను స్వీకరించాలా లేదా పాతదానితో కొనసాగించాలా అనే నిర్ణయాన్ని తీసుకోవడానికి సహాయపడుతుంది, ఇది ఆలోచనలపై ఆధారపడకుండా డేటా ఆధారిత నిర్ణయాలను నిర్ధారిస్తుంది.

    డేటాసెట్: Udacity e‑commerce A/B test (ab_data.csv).
    లింక్: https://github.com/beery4010/Analyze-AB-Test-Results/blob/master/ab_data.csv

    చరాలు: group ∈ {control, treatment}, converted ∈ {0,1}.

    లక్ష్యం: పాత (control) మరియు కొత్త (treatment) పేజీల మధ్య కన్వర్షన్ రేట్లు వేరుగా ఉన్నాయా అని పరీక్షించండి.

    హైపోథసిస్ (two‑sided):
    - $ H_0: p_{\text{treatment}} = p_{ ext{control}} $
    - $ H_1: p_{\text{treatment}} \neq p_{ ext{control}} $

    అనుమానాలు: స్వతంత్ర Bernoulli ట్రయల్స్; పెద్ద నమూనా పరిమాణాలు కాబట్టి సాధారణ సమీపం ఉంటుంది.

    def plot_z_hypothesis(
    data_list,
    pop_mean=0.0,
    pop_sd=1.0,
    alternative='two.sided',   # 'two.sided' | 'greater' | 'less'
    type_test = 'mean', #'mean' | 'prob'
    alpha=0.05,
    label='Sampling distribution',
    title='z-test (sampling distribution of x̄)',
    x_label = "Sampling distribution of x̄",
    figsize=(8, 5)
):
    """
    Visualize z-test decision regions for the sampling distribution of the sample mean.

    Parameters
    ----------
    data_list : array-like
        Sample values (used only to compute x̄ and n).
    pop_mean : float
        Hypothesized population mean (μ under H0).
    pop_sd : float
        Known population standard deviation (σ).
    alternative : str
        'two.sided', 'greater', or 'less' (same semantics as your R code).
    alpha : float
        Significance level for critical regions.
    label : str
        Label for the distribution curve.
    title : str
        Plot title.
    figsize : tuple
        Figure size.

    Returns
    -------
    fig, ax : matplotlib Figure and Axes
    """
    x = np.asarray(data_list)
    n = len(x)
    xbar = np.mean(x)

    # Standard error of the mean
    se = pop_sd / np.sqrt(n)

    # Range for plotting ±4 SE around μ (as in your R function)
    grid = np.linspace(pop_mean - 4 * se, pop_mean + 4 * se, 4001)
    pdf = stats.norm.pdf(grid, loc=pop_mean, scale=se)

    # Compute critical cutoffs under H0 for the chosen alternative
    if alternative == 'two.sided':
        # symmetric cutoffs: (alpha/2) and (1 - alpha/2)
        lower_cut = stats.norm.ppf(alpha / 2, loc=pop_mean, scale=se)
        upper_cut = stats.norm.ppf(1 - alpha / 2, loc=pop_mean, scale=se)

        # Retain between cutoffs; reject outside
        retain_mask = (grid >= lower_cut) & (grid <= upper_cut)
        reject_mask = ~retain_mask

    elif alternative == 'greater':
        # reject on right tail
        cutoff = stats.norm.ppf(1 - alpha, loc=pop_mean, scale=se)
        retain_mask = (grid <= cutoff)
        reject_mask = (grid > cutoff)
        lower_cut, upper_cut = None, cutoff

    elif alternative == 'less':
        # reject on left tail
        cutoff = stats.norm.ppf(alpha, loc=pop_mean, scale=se)
        retain_mask = (grid >= cutoff)
        reject_mask = (grid < cutoff)
        lower_cut, upper_cut = cutoff, None
    else:
        raise ValueError("alternative must be one of {'two.sided','greater','less'}")

    # Build a DataFrame like the R pipeline for convenience (optional)
    df = pd.DataFrame({'x': grid, 'pdf': pdf, 'retain': retain_mask})

    # Plot
    fig, ax = plt.subplots(figsize=figsize)
    ax.plot(df['x'], df['pdf'], color='black', lw=1.2, label=label)

    # Shade retain region
    ax.fill_between(df['x'], 0, df['pdf'], where=df['retain'], color='#69b3a2', alpha=0.4, label='Retain H₀')

    # Shade reject region(s)
    ax.fill_between(df['x'], 0, df['pdf'], where=~df['retain'], color='#e76f51', alpha=0.4, label='Reject H₀')

    # Critical lines
    if alternative == 'two.sided':
        ax.axvline(lower_cut, color='#e76f51', ls='--', lw=1)
        ax.axvline(upper_cut, color='#e76f51', ls='--', lw=1)
        ax.text(lower_cut, ax.get_ylim()[1]*0.3, f"Lower crit\n{lower_cut:.2f}", ha='right', va='top', fontsize=9)
        ax.text(upper_cut, ax.get_ylim()[1]*0.3, f"Upper crit\n{upper_cut:.2f}", ha='left', va='top', fontsize=9)
    elif alternative == 'greater':
        ax.axvline(upper_cut, color='#e76f51', ls='--', lw=1)
        ax.text(upper_cut, ax.get_ylim()[1]*0.9, f"Crit\n{upper_cut:.2f}", ha='left', va='top', fontsize=9)
    elif alternative == 'less':
        ax.axvline(lower_cut, color='#e76f51', ls='--', lw=1)
        ax.text(lower_cut, ax.get_ylim()[1]*0.9, f"Crit\n{lower_cut:.2f}", ha='right', va='top', fontsize=9)

    # x̄ line and annotation
    if(type_test == 'prob'):
        ax.axvline(xbar, color='#264653', lw=2, ls='-', label='z')
        ax.annotate(f"z = {xbar:.2f}",
                xy=(xbar, stats.norm.pdf(xbar, loc=pop_mean, scale=se)),
                xytext=(xbar, ax.get_ylim()[1]*0.6),
                arrowprops=dict(arrowstyle='->', color='#264653'),
                ha='center', color='#264653')
    else:
        ax.axvline(xbar, color='#264653', lw=2, ls='-', label='Sample mean (x̄)')
        ax.annotate(f"x̄ = {xbar:.2f}",
                xy=(xbar, stats.norm.pdf(xbar, loc=pop_mean, scale=se)),
                xytext=(xbar, ax.get_ylim()[1]*0.6),
                arrowprops=dict(arrowstyle='->', color='#264653'),
                ha='center', color='#264653')

    ax.set_title(title)
    ax.set_xlabel(x_label)
    ax.set_ylabel("Density")
    ax.legend(loc='upper right', frameon=False)
    ax.grid(alpha=0.15)
    plt.show();
plot_z_hypothesis([z_stat], type_test = 'prob', title='Two proportion z-test', label = 'Standard Normal', x_label = '')

    Welch’s t‑Test : Effect of Vitamin C on Tooth Growth¶

    ఈ క్లాసిక్ డేటాసెట్ గినియా పిగ్స్‌లో విటమిన్ C ప్రభావాన్ని అన్వేషిస్తుంది, ఇది పోషక శాస్త్రంలో ఒక ప్రాథమిక ప్రయోగం. ప్రతిస్పందన చరాలు odontoblasts యొక్క పొడవు, ఇవి దంత అభివృద్ధికి బాధ్యమైన కణాలు. 60 గినియా పిగ్స్‌ను రెండు డెలివరీ పద్ధతులలో విటమిన్ C అందించడానికి యాదృచ్ఛికంగా కేటాయించారు:
    1. Orange Juice (OJ)
    2. Ascorbic Acid (VC) (విటమిన్ C యొక్క సింథటిక్ రూపం)

    ప్రతి జంతువుకు 0.5 mg/day, 1 mg/day, లేదా 2 mg/day వంటి మూడు డోసు స్థాయిలలో ఒకటి కూడా ఇవ్వబడింది. ఈ ప్రయోగం డెలివరీ పద్ధతి దంత అభివృద్ధిని ప్రభావితం చేస్తుందా అని నిర్ణయించడానికి ఉద్దేశించబడింది, డోసేజ్ను నియంత్రించడం.

    ఎందుకు ఇది ముఖ్యమైనది:
    విటమిన్ C యొక్క వివిధ మూలాల ప్రభావితత్వాన్ని అర్థం చేసుకోవడం ఆహార సిఫార్సుల మరియు సప్లిమెంట్ ఫార్ములేషన్లను మార్గనిర్దేశం చేయవచ్చు. ఆధునిక విశ్లేషణలో, ఈ రకమైన పరీక్షలు ఆరోగ్య సంరక్షణలో రెండు చికిత్సలు లేదా జోక్యాలను పోల్చడం లేదా ఉత్పత్తి డిజైన్‌లో A/B టెస్టింగ్‌కు సమానంగా ఉంటాయి.

    \(H_1\): రెండు పద్ధతుల మధ్య దంత పొడవు సగటు వేరుగా ఉంది.

# Cohen's d (using pooled SD with group sizes)
def cohens_d(x, y):
    nx, ny = len(x), len(y)
    sx2, sy2 = np.var(x, ddof=1), np.var(y, ddof=1)
    sp2 = ((nx-1)*sx2 + (ny-1)*sy2) / (nx+ny-2)
    d = (np.mean(x) - np.mean(y)) / np.sqrt(sp2)
    return d

d = cohens_d(oj, vc)

print(f"Mean's: OJ = {np.mean(oj):.3f}, VC = {np.mean(vc):.3f}")
print(f"Cohen's d = {d:.3f}")

    One‑Way ANOVA : Branch‑wise Gross Income (with Tukey HSD)¶

    ఈ డేటాసెట్ మయన్మార్‌లో పనిచేస్తున్న ఒక సూపర్‌మార్కెట్ చైన్ నుండి సవివరమైన లావాదేవీ రికార్డులను క్యాప్చర్ చేస్తుంది, మూడు ప్రధాన నగరాలను కవర్ చేస్తుంది: యాంగోన్, నయ్పిడావ్, మరియు మాండలే. డేటా 2019 జనవరి నుండి మార్చి వరకు మూడు నెలల కాలాన్ని కవర్ చేస్తుంది, రిటైల్ ఆపరేషన్ల యొక్క సమృద్ధిగా వీక్షణను అందిస్తుంది. ప్రతి రికార్డులో శాఖ స్థానం, ఉత్పత్తి లైన్, కస్టమర్ ప్రజా గణన, చెల్లింపు పద్ధతులు, మరియు గ్రాస్ ఆదాయం మరియు మొత్తం అమ్మకాల వంటి ఆర్థిక మెట్రిక్‌లపై సమాచారం ఉంటుంది.
    మా హైపోథసిస్ టెస్ట్ కోసం, మేము శాఖ స్థానం గ్రాస్ ఆదాయాన్ని ప్రభావితం చేస్తుందా అని కేంద్రీకృతమవుతున్నాము, ఇది లాభదాయకతకు కీలకమైన మెట్రిక్. రిటైల్ మేనేజర్లు కొన్ని శాఖలు నిరంతరం ఇతరులను మించినాయా అని తెలుసుకోవడానికి తరచుగా అవసరం, ఎందుకంటే ఈ అవగాహన వనరుల కేటాయింపు, మార్కెటింగ్ వ్యూహాలు, మరియు ఇన్వెంటరీ ప్లానింగ్‌ను మార్గనిర్దేశం చేయవచ్చు.
    హైపోథసిస్:
    \(H_0\): మూడు శాఖల (A, B, C) మధ్య సగటు గ్రాస్ ఆదాయం ఒకేలా ఉంది.
    \(H_1\): కనీసం ఒక శాఖకు వేరుగా ఉన్న సగటు గ్రాస్ ఆదాయం ఉంది.

    ఎందుకు ఇది ముఖ్యమైనది: పరీక్ష ముఖ్యమైన తేడాలను వెల్లడిస్తే, నిర్వహణ కస్టమర్ కొనుగోలు శక్తి, శాఖ పరిమాణం, లేదా స్థానిక మార్కెటింగ్ ప్రభావితత్వం వంటి మూలకాలను పరిశీలించవచ్చు. ఈ విశ్లేషణ డేటా ఆధారిత నిర్ణయాలను ఆప్టిమైజ్ చేయడానికి వ్యాపార మేధస్సు పనులను ప్రతిబింబిస్తుంది.

    ఫలితాల వివరణ: ANOVA నడిపించిన తర్వాత, p-value 0.05 కంటే తక్కువ అయితే, మేము null hypothesis ను తిరస్కరించి, శాఖ స్థానం గ్రాస్ ఆదాయాన్ని ప్రభావితం చేస్తుందని తేలిస్తాము. Tukey HSD ఉపయోగించి పోస్ట్-హోక్ విశ్లేషణ ముఖ్యంగా వేరుగా ఉన్న శాఖలను గుర్తిస్తుంది, మెరుగుదల కోసం లక్ష్యమైన వ్యూహాలను సాధించడానికి వీలు కల్పిస్తుంది.

    - gender ∈ {Male, Female}
    - category ∈ {ఉదాహరణకు, Clothing, Electronics, Accessories, …} (ఫైల్‌లో అనేక కేటగిరీలు ఉన్నాయి)

    హైపోథసిస్ (Chi‑Square Test of Independence):
    \(H_0\): లింగం మరియు ఉత్పత్తి కేటగిరీ స్వతంత్రంగా ఉన్నాయి (ఏ అసోసియేషన్ లేదు).
    \(H_1\): లింగం మరియు ఉత్పత్తి కేటగిరీ అనుసంధానితంగా ఉన్నాయి (కేటగిరీ ప్రాధాన్యత లింగంపై ఆధారపడి ఉంది).

    ఎందుకు ఇది ముఖ్యమైనది:
    ముఖ్యమైన అనుసంధానం లింగం ద్వారా వేరుగా ఉన్న కేటగిరీ ప్రాధాన్యతలను సూచిస్తుంది. రిటైలర్లు ప్రమోషన్లను, కంటెంట్, ఇన్వెంటరీ మిక్స్, మరియు స్టోర్ డిస్ప్లేలను డిమాండ్‌ను మెరుగ్గా సరిపోల్చడానికి సర్దుబాటు చేయవచ్చు, ఇది తరచుగా కన్వర్షన్ రేట్లు మరియు గ్రాస్ మార్జిన్‌ను మెరుగుపరుస్తుంది.

    అనుమానాలు & డేటా తనిఖీలు:
    - సరిపోతున్న అంచనాలు (అనుకూలంగా ≥ 5 ప్రతి సెల్). కొన్ని కేటగిరీలు అరుదుగా ఉంటే, సమాన కేటగిరీలను సమూహీకరించడం లేదా అంచనాలను సంతృప్తి పరచడానికి టాప్ N కేటగిరీలను విశ్లేషించడం పరిగణనలోకి తీసుకోండి.
    - గమనికలు స్వతంత్రంగా ఉంటాయి (ప్రతి వరుస ఒక ప్రత్యేక లావాదేవీ).
    (ఇవి కేటగిరీ విశ్లేషణలో Chi‑Square పరీక్షకు సాధారణ పరిస్థితులు.)

    డేటాసెట్ మూలం:
    Customer Shopping Data (GitHub) — ఫైల్: customer_shopping_data.csv

    అర్థం:
    1. If p < 0.05, reject \(H_0\): లింగం మరియు ఉత్పత్తి కేటగిరీ అనుసంధానితంగా ఉన్నాయి.
    2. Cramér’s V అనుసంధాన శక్తిని సూచిస్తుంది: ~0.1 = బలహీనమైనది, ~0.3 = మోస్తరు, ~0.5 = బలమైనది.

    మేము Chi‑square test of independence మరియు Cramér’s V ను ప్రభావ పరిమాణంగా లెక్కిస్తాము.

    # Build contingency table: Gender vs Payment Method
ct = pd.crosstab(df['category'], df['gender'])
chi2, p, dof, expected = chi2_contingency(ct)
print("Contingency Table:\n", ct)
print(f"Chi-square = {chi2:.4f}, p-value = {p:.6f}, dof = {dof}")

    # Compute Cramér's V
n = ct.values.sum()
phi2 = chi2 / n
r, c = ct.shape
cramers_v = np.sqrt(phi2 / min(r-1, c-1))

print(f"Cramér's V = {cramers_v:.3f}")

    Cramér's V = 0.009

    0.009 చాలా బలహీనమైన అనుసంధానాన్ని సూచిస్తుంది

    from statsmodels.graphics.mosaicplot import mosaic
def plot_mosaics(data, x_col, y_col, title='', colors_list =[]):
    dict_of_tuples = {}

    # create the clean set of percentages to print
    for x_col_ in data[x_col].unique():
        for y_col_ in data[y_col].unique():
            n = len(data[(data[x_col]==x_col_)&(data[y_col]==y_col_)][x_col])
            d = len(data[(data[x_col]==x_col_)][x_col])
            len_ = len(data[x_col])
            if((d==0) or (n/d<=0.04)):
                # if the percentage within a class is less than 4%, do not print the percentage
                dict_of_tuples[(str(x_col_), str(y_col_))] = ''
            elif(n/len_<=0.02):
                # If its a tiny class with less than 2% of the total data, do not print
                dict_of_tuples[(str(x_col_), str(y_col_))] = ''
            else:
                dict_of_tuples[(str(x_col_), str(y_col_))] = str(int(n/d*100))+"%"

    dict_of_colors = dict_of_tuples.copy()
    if(len(colors_list)>0):
        # create a clean set of colors
        for i, x_col_ in enumerate(data[x_col].unique()):
            for y_col_ in data[y_col].unique():
                dict_of_colors[(str(x_col_), str(y_col_))] = {'color':colors_list[i], 'alpha':0.8}

    # Plot the mosaic plot
    labelizer = lambda k: dict_of_tuples[k]    
    fig, ax = plt.subplots(figsize=(8,6))
    if(len(colors_list)>0):
        mosaic(data.sort_values([x_col, y_col]), [x_col, y_col], 
               statistic = False, axes_label = True, label_rotation = [90, 0],
               labelizer=labelizer, properties=dict_of_colors, gap=0.008, ax=ax)
    else:
        mosaic(data.sort_values([x_col, y_col]), [x_col, y_col], 
               statistic = False, axes_label = True, label_rotation = [90, 0],
               labelizer=labelizer, gap=0.008, ax=ax)
    if(title==''):
        plt.title(str(y_col) + ' percentages across ' + str(x_col))
    else:
        plt.title(title)
    plt.show();
plot_mosaics(df, 'category', 'gender')
